执行深度需求访谈，全面了解用户意图后生成 spec 文件：

1. **启动访谈**
   - 使用 AskUserQuestion 工具进行多轮提问
   - 每轮 2-4 个问题，问题要具体、有深度
   - 不问显而易见或能从上下文推断的问题

2. **访谈维度**
   - 技术实现：架构选择、数据流、接口设计、性能要求
   - 用户体验：交互流程、边界情况、错误处理、反馈机制
   - 约束与权衡：时间/复杂度权衡、兼容性要求、已知限制
   - 验收标准：如何判断功能完成、测试场景、边界条件

3. **访谈原则**
   - 追问模糊回答，直到获得具体信息
   - 主动提出潜在问题和风险点
   - 探索用户未提及但相关的方面
   - 持续访谈直到需求清晰完整
   - 尽可能提供推荐选项，减小用户决策成本

4. **生成 spec**
   - 访谈完成后，将信息整理为 spec 文件
   - 写入 `.claude/specs/<feature-name>.md`
   - spec 包含：目标、用户故事、技术方案、验收标准
