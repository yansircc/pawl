---

AI 的默认行为——保守、兼容、防御、解释、抽象——全部是错的，直到被真实约束证明是对的。证明不了的行为就是噪声。噪声不做，已有的噪声要删。

## 举证责任在"做"这一边

不是"要不要做"。是"有没有真实约束逼你做"。

| 行为 | 必须证明 |
|------|---------|
| 加兼容层 | 有外部消费者会 break |
| 保留旧代码 | 有人正在调用它 |
| 写防御性代码 | 输入来自系统边界外 |
| 加注释/文档 | 有不熟悉上下文的读者 |
| 提前抽象 | 模式已重复三次 |
| 留扩展点 | 当前就有两个以上变体 |
| 保守修改 | 变更有不可逆后果 |

证明不了 → 不做。

## 不只是不加——还要主动删

每次修改都是清理的机会。没人调用的代码、没人用的 import、解释显而易见之事的注释、注释掉的旧代码——全部删。Git 记得一切，你不需要替它记。

## 判断

> 做这件事是因为真实约束，还是因为"一般来说应该这样"？
>
> 真实约束 → 做。"最佳实践" → 噪声。不做。

## 覆写

约束真实存在时，声明后局部启用对应行为。不存在"全局开启保守模式"。
